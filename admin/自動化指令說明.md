# 自動化指令說明

## 🚀 快速開始

系統已配置完整的自動化測試、檢查、修復流程。只需執行一個指令即可完成所有檢查：

```bash
npm run auto
```

## 📋 可用指令

### 1. 完整自動化流程（推薦）
```bash
npm run auto
```
執行完整的自動化流程：
- ✅ 自動修復常見問題
- ✅ 自動測試所有功能
- ✅ 代碼質量檢查
- ✅ 建置驗證

### 2. 自動測試
```bash
npm run test
```
檢查：
- 必要檔案是否存在
- 核心組件是否完整
- 建置是否成功
- 代碼語法是否正確

### 3. 代碼質量檢查
```bash
npm run check
```
檢查：
- 代碼質量問題
- 潛在錯誤
- 最佳實踐
- 建置輸出大小

### 4. 自動修復
```bash
npm run fix
```
自動修復：
- package.json scripts
- .gitignore 配置
- 目錄結構
- 其他常見問題

## 🎯 使用場景

### 開發前檢查
```bash
npm run auto
```
確保所有功能正常，代碼質量良好。

### 提交前檢查
```bash
npm run check
```
檢查代碼質量，確保沒有問題。

### 遇到問題時
```bash
npm run fix
npm run test
```
先自動修復，再測試驗證。

## 📊 輸出說明

### ✅ 通過
- 綠色 ✓ 標記表示通過
- 所有檢查都通過時，系統會顯示成功訊息

### ⚠️ 警告
- 黃色 ⚠ 標記表示警告
- 不影響功能，但建議修復

### ✗ 錯誤
- 紅色 ✗ 標記表示錯誤
- 需要立即修復

## 🔧 自定義檢查

所有檢查腳本位於 `scripts/` 目錄：
- `auto-test.js` - 測試腳本
- `auto-check.js` - 檢查腳本
- `auto-fix.js` - 修復腳本
- `auto.js` - 主整合腳本

可以根據需要修改這些腳本。

## 💡 提示

1. **定期執行**：建議每次修改代碼後執行 `npm run auto`
2. **CI/CD 整合**：可以將 `npm run auto` 整合到 CI/CD 流程中
3. **快速檢查**：開發過程中可以使用 `npm run check` 快速檢查
